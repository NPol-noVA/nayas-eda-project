# My-Submission-Checklist



## Intro:  

 It is very important to have a checklist before submitting any project. Sometimes a project is complete and works correctly, but reviewers cannot access it because the link is broken, returns a 404 error, or requires permission. In those cases, the work cannot be reviewed and may be treated as if it was not submitted.
 To avoid this problem, I will use the following checklist before every submission.


## My Checklist:

* [ ] Repository/document name matches what was requested.
* [ ] Files use relative paths (no `C:\Users\...`).
* [ ] README is complete and up to date.
* [ ] Repository or document is public and accessible to anyone with the link.
* [ ] The link opens successfully in a private/incognito window (no 404 errors and no access request).
