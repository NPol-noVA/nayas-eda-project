## one-task-one-repository




## introduction:
  each project need its own repository to guarantee that it is organized, clear, easy to read and can be improved. this is why we are re-creating repositories and learning how to organize our own template and 
  unique structure that we will follow for next and future tasks. it is kind of personalization.  



## how to run it: 
  it is very easy because it is really nothing, this repository is just to learn how to organize so there is no specific things about it. in general, there must be a folder for a data set,notebook/sript, paths 
  and readme file so:
  
  1.install required paths and tools
  2.run the script on your kernel 
  3.read readme file for guidance
  4.collaborating with pull requests if avaialable or try to suggest improvements or make your own branches 



 ## a data-set:
   this is real data-set installed from a governmental site and it is clear and ready to use. it is talking about the amount of insurance charges and its relationships with factors such as age, sex, bmi  
   information, number of children and region, with each separated in a separate column and has its own records. this is a clear data-set as i cited before so it is organized, has no empty cells, has no wrong 
   data,etc.
